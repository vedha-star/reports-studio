Webpack 5 / Vite Configuration
Complete setup for both the remote (Report Studio) and the host app.
💡
Cookie auth means zero auth config in webpack
Because authentication uses a shared .navacle.io cookie, neither the remote nor the host needs to pass tokens, configure interceptors, or share auth state. The browser attaches the cookie to every API call automatically.
1
Remote — Report Studio (webpack.config.js)
Report Studio exposes its modules. The filename is the entry point host apps will reference.
webpack.config.js · studio.navacle.io
const { ModuleFederationPlugin } = require('webpack/lib/container/ModuleFederationPlugin') module.exports = { // ... other config plugins: [ new ModuleFederationPlugin({ name: 'navacle_rs', filename: 'remoteEntry.js', exposes: { // Full run widget with params, format picker, generate './ReportRunner': './src/mfe/ReportRunner', // Searchable list of all reports for the user './ReportBrowser': './src/mfe/ReportBrowser', // Minimal inline download button './DownloadButton': './src/mfe/DownloadButton', }, shared: { react: { singleton: true, requiredVersion: '^18.0.0' }, 'react-dom': { singleton: true, requiredVersion: '^18.0.0' }, }, }), ], }
2
Host App — Admissions Portal (webpack.config.js)
The host declares the remote URL. This is the only place the studio URL appears — no auth token, no API key.
webpack.config.js · app.navacle.io
new ModuleFederationPlugin({ name: 'admissions_portal', remotes: { // Key used in imports: import('navacle-report-studio/...') 'navacle-report-studio': 'navacle_rs@https://studio.navacle.io/remoteEntry.js', }, shared: { react: { singleton: true, requiredVersion: '^18.0.0' }, 'react-dom': { singleton: true, requiredVersion: '^18.0.0' }, }, })
3
Using ReportRunner in a host page
React.lazy + Suspense for code splitting. The widget resolves auth from the cookie — no token prop needed.
EnquiryPage.tsx · host app
import React from 'react' // Lazy load — bundle only fetched when component mounts const ReportRunner = React.lazy( () => import('navacle-report-studio/ReportRunner') ) export default function EnquiryPage() { return ( <div className="page-layout"> <EnquiryTable /> {/* MFE widget — auth from cookie, no token needed */} <React.Suspense fallback={<ReportSkeleton />}> <ReportRunner reportSlug="submitted-enquiries" apiBaseUrl="https://api.navacle.io" {/* No authToken prop — resolved from .navacle.io cookie */} defaultFormat="xlsx" onSuccess={fileName => toast.success(`Downloaded: ${fileName}`) } onError={err => toast.error(err.message) } /> </React.Suspense> </div> ) }
4
Vite alternative (vite.config.ts)
Same pattern using @originjs/vite-plugin-federation for Vite-based host apps.
vite.config.ts
import federation from '@originjs/vite-plugin-federation' export default { plugins: [ federation({ name: 'admissions_portal', remotes: { 'navacle-report-studio': 'https://studio.navacle.io/assets/remoteEntry.js', }, shared: ['react', 'react-dom'], }), ], build: { target: 'esnext' }, }
⚠ CORS configuration required: studio.navacle.io must respond with Access-Control-Allow-Origin: https://app.navacle.io (and other host origins) and Access-Control-Allow-Credentials: true for the cookie to be forwarded.
Cookie Authentication Flow
How a single shared session cookie handles auth across all subdomains — including the MFE widget.
✅ What cookie auth gives you
• No token management in JS
• No Authorization headers to configure
• Automatic RBAC — API always sees real user
• Works across all Navacle subdomains automatically
• Session expiry handled in one place
• XSS-safe (HttpOnly prevents JS theft)
⚠ Requirements
• All apps must be on *.navacle.io subdomains
• CORS must allow credentials: credentials: "include"
• api.navacle.io must set Access-Control-Allow-Credentials: true
• Cannot use wildcard * in CORS origin with credentials
• Third-party embeds outside .navacle.io need token fallback
